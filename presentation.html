PART 1

PI Agent란 무엇인가?

PI(Personal Intelligence)는 LLM 기반 자율 AI 에이전트를 구축하기 위한
TypeScript 툴킷으로, OpenClaw의 핵심 런타임 엔진입니다.

📚

pi-ai

LLM 추상화 레이어
Model, streamSimple
멀티 프로바이더

⚙

pi-agent-core

에이전트 루프
도구 실행 엔진
AgentMessage 타입

💻

pi-coding-agent

고수준 SDK
SessionManager
내장 도구

🖥

pi-tui

터미널 UI
CLI 인터페이스
TUI 컴포넌트

핵심 철학: "LLM은 코드를 쓰고 실행하는 데 뛰어나다. 이를 수용하라." - 가장 짧은 시스템 프롬프트, 4개의 핵심 도구(Read, Write, Edit, Bash)만으로 동작

NANOCLAW

NanoClaw - 컨테이너 격리 경량화

Node.jsMIT "OpenClaw의 50만 줄을 이해하기 쉬운 단일 프로세스로"

🐇 5가지 핵심 원칙

미니멀리즘 - 전체 코드베이스를 이해할 수 있는 크기
격리 - 컨테이너 기반 보안 (allowlist 아닌 Docker)
개인화 - 코드 수정 기반 커스터마이징
코드 주도 - config 파일이 아닌 직접 수정
AI 네이티브 - Claude Code로 디버깅

🛠 아키텍처

agentskills.io

OpenClaw과 Hermes Agent가 공유하는 오픈 스킬 표준.
스킬 정의, 메타데이터, 실행 인터페이스를 통일하여
프레임워크 간 스킬 호환성을 보장합니다.

🌐

OGP (Open Gateway Protocol)

서로 다른 프레임워크의 에이전트가 암호화 서명된 메시지를
교환하는 연합(Federation) 레이어.
크로스 프레임워크 에이전트 통신을 가능하게 합니다.

🔌

MCP (Model Context Protocol)

PicoClaw이 네이티브로 지원하는 도구 연결 표준.
어떤 MCP 서버든 연결하여 에이전트에
새로운 능력을 부여할 수 있습니다.

🔐

Gateway WebSocket

OpenClaw의 ws://127.0.0.1:18789 제어 평면.
세션, 채널, 도구, 이벤트를 단일 포인트에서
조율하는 로컬 우선 아키텍처.
